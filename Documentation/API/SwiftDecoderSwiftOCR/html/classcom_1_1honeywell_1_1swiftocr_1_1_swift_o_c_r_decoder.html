int com.honeywell.swiftocr.SwiftOCRDecoder.addOCRTemplate ( byte[]  imagedata)

Fetch's the template information shared as image data array then map it to the template ID and return the newly created templateID. The symbology should be set to QR prior to using this API.

Parameters
imagedataByte Data for the given Template
Returns
templateId of the newly added template.
If the template is duplicate -2 is returned.
If template count is more than 40 then -3 is returned.
if any error occurred -1 is returned.

◆ addResultListener()

void com.honeywell.swiftocr.SwiftOCRDecoder.addResultListener ( SwiftOCRResultListener  resultListener)

This method adds an TemplateOCRResultListener that will be notified of decode results.

By adding a listener you are implicitly registering a callback behind the scenes with the system. Any class that wishes to receive decode results must implement SwiftOCRResultListener. If you would like to use your own plug-ins and only your own plug-ins, then you should not add your self as a listener here but should do so within your own custom Swift plug-in.

Parameters
resultListenerThe TemplateOCRResultListener that will receive decode results on a decode event.

SwiftOCRStatus com.honeywell.swiftocr.SwiftOCRDecoder.enableSwiftOCRFeature ( boolean  enable)

Enable and disable SwiftOCR feature based on argument It also checks whether swiftocr Microservice is enabled for the the given license or not.

Parameters
enableset true to enable swiftocr feature and false to disable
Returns
SUCCESS if SwiftOCR feature is available for the given license. FEATURE_NOTAVAILABLE if SwiftOCR feature is not available for the given license. FAILED if any exception due to network. SWIFTOCR_DETECTION_MODE_NOT_SET If the required detection mode is not set prior to calling this API.
