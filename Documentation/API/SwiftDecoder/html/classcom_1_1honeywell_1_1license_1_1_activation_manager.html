Detailed Description

This class is responsible for managing the activation and deactivation of SwiftDecoder. The API must be activate before any useful functionality can be used. Please note that any settings changes made to HSMDecoder before activation has occurred will be wiped during the activation process and must be reset.

Get the device ID.

SwiftDecoder is using secure ID as default device ID because the license model is per device. If the user chooses to use serial number, call useSerialnumber API as the same would be used as device ID. This function can be used to get the ID of one device and check the activation status from the Honeywell portal. However, it must not be used for any other purpose as this one. The Android developer site forbids using hardware identifiers for advertising for example. More information here: https://developer.android.com/training/articles/user-data-ids

◆ useSerialNumber()

static ActivationResult com.honeywell.license.ActivationManager.useSerialNumber ( Context  context)
static

This API enables to use the serial number as unique ID for license activation. Needs to be used with caution, only if the native SwiftDecoder SDK way of license activation is not working. Also if using serial number should not switch between serial number and other ID on one device as license is tied up to specific ID As fetching serial number app needs sufficient permissions(READ_PRIVILEGED_PHONE_STATE ), the application should be provided with those permissions. Only if this call returns true SDK can use serial number for license activation. If this call returns false, SDK will use native way of fetching unique ID Once this call returns true for debugging can use getDeviceId to see what is the serial number used.

Parameters
contextApplication context
Returns
SUCCESS if SDK is able to fetch the serial number FAILED_FETCHING_SERIAL_ID if SDK fails to fetch the serial number (Check app permissions)
