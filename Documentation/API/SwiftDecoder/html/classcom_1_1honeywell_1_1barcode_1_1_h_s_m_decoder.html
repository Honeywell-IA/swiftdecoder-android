void com.honeywell.barcode.HSMDecoder.addResultListener ( DecodeResultListener  resultListener)

This method adds an DecodeResultListener that will be notified of decode results.

By adding a listener you are implicitly registering the Default Decode plug-in behind the scenes with the system. Any class that wishes to receive decode results must implement OnDecodeResultListener. Accepts 8bpp grayscale image pixel data and returns the decoded results. This should be raw pixel data and not contain any image header data. 

@param imgData

The raw 8bpp grayscale image pixel data

Parameters
imgWidthThe pixel width of the image
imgHeightThe pixel height of the image
imgLineDeltaThe number of bytes between two consecutive rows of pixels. In most cases it is the image width itself but sometimes it might be different because of extra padding. In all cases, imgLineDelta = imgWidth + padding. Padding is generally used so that each line start at an aligned memory address (multiples of 2, 4, etc.) This padding is sometimes also known as "row stride" or "line stride"
Returns
An array of decode results

◆ keepCameraInitialized()

void com.honeywell.barcode.HSMDecoder.keepCameraInitialized ( boolean  enable)

This method controls whether or not the camera stays initialized in between decode attempts.

Setting this to false will cause the camera to be released after each decode. This means that it will also need to be re-initialized each time as well which will degrade overall decode time. If set to true, the camera will be kept initialized for the life of the HSMDecoder object. This means that the camera will be locked for use from other applications until HSMDecoder.disposeInstance() is called. Developers must weigh which is more important, speed or camera flexibility. The camera stays initialized by default.

Parameters
enableDetermines whether or not the camera will stay initialized between decode attempts
